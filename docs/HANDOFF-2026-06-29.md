## 0. 项目初衷（新窗口必读，别跑偏）

**定位**：tcm-db = 中医领域的「**知识编译器 + 证据数据库**」。
- ❌ 不是 SQLite CRUD 网站 / 不是全文搜索器 / 不是让 LLM 直接下诊断的系统
- ✅ 是把倪海厦散落资料清洗、结构化、来源可追溯地沉淀，**既服务人类浏览，也为 LLM 编译高质量上下文**

**灵感**：Karpathy LLM Knowledgebase（6 原则：持续整理非临时搜索 / 清洗结构化沉淀 / 来源可追溯 / 服务人类+LLM / 知识质量>基础设施 / 持续增量更新感知失效重复）。

**目标推理链**：`症状 → 证型 → 方剂 → 药物/医案证据`（可解释、每步可溯源）。

**硬约束**：纯 Python 标准库 + SQLite + http.server；**不上向量库/微服务/ORM**；中医补全**必须有原文证据，禁模型常识写库**；不输出未经限定的临床诊断。

**当前推理链断点**（待补）：症状→证型 89%✓ → **证型→方剂 6.2% 致命断** → 方→药 26% → 案→方 17%。这是后续方证补全的核心动机。
